# 实验三：Linux进程管理

## 项目结构
- shell/               模拟Shell
- pipe_comm/           管道通信
- msg_queue/           消息队列通信
- shared_memory/       共享内存通信
- docs/                文档与验收材料
- common/              公共头文件

## 分工原则
采用“模块负责人 + 综合支撑 + 全员交叉学习”的方式完成实验。

## 协作要求
1. 每个人都必须参与技术内容
2. 每个人都必须掌握四个实验的基本思路
3. 每个人都要能接受随机抽查
4. 谁负责哪个文件，谁就负责讲清该文件
