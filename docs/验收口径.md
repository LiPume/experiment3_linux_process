# 验收口径

我们小组共有6人，采用“模块负责人 + 综合支撑 + 交叉学习”的方式完成实验。

其中：
- 1号负责整体统筹、代码整合、组内讲解组织，并辅助模拟Shell部分；
- 2号主负责模拟Shell的实现及 fork()、execl()、wait() 原理分析；
- 3号主负责管道通信程序及阻塞读写分析；
- 4号主负责消息队列线程通信及同步机制设计；
- 5号主负责共享内存进程通信及互斥同步实现；
- 6号主负责测试验证、文档整理、答辩材料汇总，并辅助技术模块分析。

在此基础上，我们进行了交叉讲解，确保每位成员都了解四个实验的实现思路、关键系统调用和主要代码逻辑，能够接受随机验收。
